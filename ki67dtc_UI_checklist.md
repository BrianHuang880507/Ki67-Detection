## 一、整體使用流程（User Flow）

你定義的流程是：

1. 開啟程式  
2. 選輸入資料夾  
3. 載入模型  
4. 分割細胞輪廓  
5. 細胞參數分析和顯示  
6. 保存選輸出資料夾  
7. 保存分析報告 (CSV) 和 分割 NPY  

我幫你稍微「拆細 + 排序成軟體行為」，不改本質：

1. **開啟程式**
   - 程式啟動時顯示主畫面：
     - 左側是「路徑與參數區」
     - 右側是「影像顯示 + 統計區」
   - 如果有記住上一次的輸入/輸出資料夾，預先顯示在欄位中。

2. **選輸入資料夾**
   - 使用者按「選擇輸入資料夾」：
     - 選擇一個資料夾，程式掃描裡面的可用影像（PNG/JPG/TIF、多通道 TIFF）。
     - UI 左邊或右下方更新「影像清單（檔名列表）」。
   - 若資料夾為空或沒有支援的影像格式：
     - UI 顯示錯誤訊息，不允許按 Run。

3. **載入模型**
   - 使用者按「載入模型」：
     - 選擇一個模型權重檔（例如 `.pt` / `.pth` / 你專案實際用的格式）。
   - UI 顯示：
     - 目前已載入模型的檔名（以及簡短資訊：例如訓練日期/版本，如果拿得到）。
   - 若模型載入失敗：
     - UI 底部顯示錯誤訊息。
     - 不允許開始分割。

4. **設定推論與運算方式（隱含在 3~4 之間）**
   - 在開始分割之前，使用者可以調整：
     - 閾值 / confidence threshold
     - 最小/最大細胞大小
     - 通道選擇（核、Ki67 通道等）
     - CPU / GPU 模式（並清楚顯示目前狀態）
   - 這些參數在 Run 之後會套用到「整個資料夾的所有影像」。

5. **分割細胞輪廓（批次 segmentation）**
   - 使用者按下「Run」：
     - 程式依序對輸入資料夾裡的所有影像做 segmentation。
     - 中途 UI 顯示：
       - 進度列：已完成 X / N
       - 可選地在訊息區顯示「目前正在處理：XXX.png」。
   - 使用者可以：
     - 按 Stop/Cancel：中止剩餘影像的處理。
     - 按 Reset：清空目前結果與狀態，可以重新來一次。

6. **細胞參數分析和顯示（逐張檢視）**
   - 分割完成後，進入「檢視結果」狀態：
     - 影像清單可供選擇哪一張要看。
     - 也可以用「上一張 / 下一張」快速切換。
   - 對於目前選中的影像，畫面顯示：
     - 原始影像。
     - segmentation mask 的 overlay（可調透明度）。
     - 可切換顯示：原圖 / overlay / mask。
     - 不同顏色區分 Ki67+ / Ki67-。
   - 當使用者在影像上「點選某個 cell」時：
     - 該 cell 高亮顯示。
     - 右側或下方顯示此 cell 的屬性（面積、強度等）。
   - 同時顯示此影像的整體統計：
     - Ki67 positive cell 數。
     - Ki67 negative cell 數。

7. **選輸出資料夾**
   - 使用者按「選擇輸出資料夾」：
     - 指定要把結果寫到哪個位置。
   - 如果這個資料夾裡已經有同名檔案（例如同名 npy/CSV）：
     - UI 彈出警告（你勾選要有覆蓋警告）。
     - 讓使用者選擇是否覆蓋或取消。

8. **保存分析報告 (CSV) 和 分割 npy**
   - 當使用者按「匯出結果」或「保存報告」：
     - 程式在輸出資料夾中：
       - 產生一個 CSV：
         - 每一列對應一張影像，包含：
           - 檔名
           - Ki67 positive 數
           - Ki67 negative 數
           -（可選）其他指標，如 Ki67 index
       - 把每張影像的 segmentation mask 存為 npy（或 tiff，視你實作）。
       - 匯出 overlay PNG（方便簡報/報告）。
   - 匯出過程中若有錯誤（例如磁碟無法寫入）：
     - UI 底部顯示錯誤摘要。
     - 提供「開啟 log 檔」協助 debug。

## 二、對應的畫面區塊（你可以想像的 UI 骨架）

結合你核准的設計，概念上可以長這樣：

### 左側：流程控制 & 參數區

1. **路徑區塊**
   - 輸入資料夾：[Browse] + 顯示路徑
   - 輸出資料夾：[Browse] + 顯示路徑

2. **模型區塊**
   - 模型路徑：[Browse] + 顯示選到的檔名
   - 顯示目前使用的是 CPU / GPU

3. **參數區塊**
   - 閾值：
   - 最小/最大 cell size：
   - 通道選擇（下拉或簡單文字選擇）

4. **執行控制區塊**
   - [Run] [Stop/Cancel] [Reset]
   - 進度列：X / N
   - [開啟 log 檔] 按鈕

---

### 右側上方：影像視窗

- Viewer 視窗：
  - 顯示目前選中的影像：
    - 模式切換按鈕：原圖 / overlay / mask
    - overlay 透明度滑桿
  - 支援縮放、平移
  - 滑鼠點選 cell → 高亮 + 通知右下方的 cell 資訊區

---

### 右側下方：影像列表 + 統計 + cell 資訊

1. **影像列表 & 瀏覽**
   - 顯示所有影像檔名清單。
   - [上一張] [下一張] 按鈕切換目前檢視對象。

2. **本張影像統計**
   - Ki67 positive count
   - Ki67 negative count

3. **選中 cell 的屬性**
   - 面積
   - 平均強度 / 其他你 pipeline 算得出來的參數

---

### 最底部：系統訊息列

- 用一行簡短文字顯示系統狀態：
  - 例如：
    - 「已完成 10/32 張影像」
    - 「ERROR: 找不到模型檔案」
    - 「輸入資料夾為空，請重新選擇」
